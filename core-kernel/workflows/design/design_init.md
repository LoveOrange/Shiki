# Design Init

> Role: Architect. Analyze the Design Brief, expand the plan, and register targets. Do not write design body content.

## Load

- `features/{feature}/design_brief.md`
- `features/{feature}/_plan.md` bootstrap plan
- `features/{feature}/index.md`
- `core-kernel/templates/feature/_plan.md`
- `shiki_context/modules/{module}/index.md` when present

## Steps

1. Read the Design Brief and estimate size, scope, and risk.
2. For structural changes, expand `_plan.md` into model, persistence, ACL, component, entrance, flow, code_contract, code, and merge tasks.
3. For tiny code-only fixes, keep the plan minimal and route to the smallest necessary tasks.
4. Create missing module overlay directories when needed.
5. Update feature index entries for generated targets.
6. Do not generate design specs or code body text.

## Output

- Update `features/{feature}/_plan.md`.
- Update `features/{feature}/index.md`.
- Update baseline module index only when baseline artifacts are affected.

## Verification

1. Plan shape matches the brief size.
2. Every contract path resolves.
3. No design body files are generated by this workflow.
4. README files are not used for runtime routing.
